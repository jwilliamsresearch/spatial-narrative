Overview

spatial-narrative is a Rust library for modeling, indexing, and analyzing events that unfold across both geographic space and chronological time. It provides efficient data structures and algorithms for working with spatiotemporal narratives.

Key Capabilities

  • Flexible Event Representation: Events with WGS84 coordinates, multi-precision timestamps (year to second), and extensible metadata
  • Spatial Indexing: R*-tree and quadtree structures for efficient spatial queries
  • Text Analysis: Extract named entities, locations, and temporal expressions from unstructured text
  • Movement Analysis: Trajectory analysis, speed calculation, stop detection
  • Multiple I/O Formats: JSON, GeoJSON, CSV, GPX, SQLite with full-text search
  • ML Integration: Optional BERT-based NER via ONNX Runtime

Installation

Add to your Cargo.toml:

TOML 
[dependencies]
spatial-narrative = "0.1"

# With optional features
spatial-narrative = { version = "0.1", features = ["full"] }

Features

Core Types

Fundamental data structures for spatiotemporal events: Location (WGS84), Timestamp (multi-precision), Event, and Narrative collections.

Spatial Indexing

R*-tree and quadtree implementations for efficient spatial queries, nearest neighbor search, and bounding box operations.

Text Processing

Pattern-based and ML-powered named entity recognition, keyword extraction, and temporal expression parsing.

Movement Analysis

Trajectory simplification, speed calculation, stop detection, and spatiotemporal clustering algorithms.

I/O Support

Read and write multiple formats: JSON, GeoJSON, CSV, GPX tracks, and SQLite databases.

Analysis Tools

DBSCAN and k-means clustering, temporal windowing, narrative comparison, and statistical summaries.

Examples

Creating Events

Rust 
use spatial_narrative::prelude::*;

// Create a location
let paris = Location::new(48.8566, 2.3522);

// Create an event
let event = Event::builder()
    .text("Arrived in Paris")
    .location(paris)
    .timestamp(Timestamp::parse("2024-03-15T10:30:00Z")?)
    .tags(vec!["travel", "europe"])
    .build()?;

// Build a narrative
let mut narrative = Narrative::new("European Journey");
narrative.add_event(event);

Spatial Queries

Rust 
use spatial_narrative::index::SpatialIndex;

let mut index = SpatialIndex::new();

// Add events to index
for event in events {
    index.insert(event);
}

// Find 5 nearest events
let point = Location::new(48.8566, 2.3522);
let nearest = index.nearest(&point, 5);

// Query by bounding box
let bounds = GeoBounds::new(
    Location::new(48.8, 2.2),
    Location::new(48.9, 2.4),
);
let in_area = index.query_bbox(&bounds);

Named Entity Recognition

Rust 
use spatial_narrative::text::TextAnalyzer;

let analyzer = TextAnalyzer::new();
let text = "Dr. Smith visited the Louvre Museum in Paris on March 15th.";

for entity in analyzer.extract_entities(text) {
    println!("{}: {}", entity.entity_type, entity.text);
}
ML-Based NER: For higher accuracy, use the optional ml-ner feature with BERT models. See the ML NER Guide for setup instructions.
